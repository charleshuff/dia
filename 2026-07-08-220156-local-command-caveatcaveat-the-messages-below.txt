     Keyboard-driven shape insertion for Dia (Python plugin)

     Context

     You want to spawn UML and Flowchart shapes from the keyboard instead of
     hunting through the tool palette, and to bulk-import a list of words as
     individual text boxes. The Dia palette exposes shapes as SheetObject
     entries clicked with the mouse; there is no built-in keyboard path from a
     palette entry to placing an object on the canvas, and no per-shape
     accelerator wiring exists in app/toolbox.c.

     Good news: Dia already has an embedded Python 3.8+ plugin system that
     auto-loads .py files from ~/.dia/python/. It exposes:
     - dia.register_action(id, desc, menupath, callback) — adds a menu item
     (which GTK will make accelerator-bindable)
     - dia.get_object_type("UML - Class").create(x, y) → (obj, h1, h2) —
     instantiates any palette shape by name
     - dia.registered_sheets() — enumerates every sheet (UML, Flowchart, …) and
     its objects for autocomplete
     - dia.active_display() / diagram.get_sorted_selected() /
     layer.add_object(obj) — placement and mutation
     - gi.repository.Gtk is importable inside a plugin (confirmed by
     dia_rotate.py, scascale.py, select_by.py), so dialogs are fine

     So a Python plugin can absolutely give you keyboard access to any palette
     shape. What it cannot do is set an accelerator programmatically —
     accelerators live in GTK's accel map (~/.config/dia/menurc). We work around
      this by exposing one dialog behind one menu item, so binding a single key
     (e.g. F5) gets you to every shape.

     Approach

     One plugin file ~/.dia/python/keyboard_insert.py registers two menu 
     actions:

     Action 1 — QuickInsertShape (menupath 
     /DisplayMenu/Objects/QuickInsertShape)

     Opens a small Gtk.Window with:
     - A Gtk.Entry with a Gtk.EntryCompletion populated from every SheetObject
     in the UML and Flowchart sheets (via dia.registered_sheets(), filter
     sheet.name in {"UML", "Flowchart"}). Completion matches on both the object
     type name ("UML - Class") and the friendly description ("Class").
     - A second Gtk.Entry for an optional label — written into
     obj.properties["name"] if present, else obj.properties["text"] if present,
     else silently skipped.
     - Enter → insert; Esc → close.

     Placement logic (in a helper _placement_point(diagram)):
     1. sel = diagram.get_sorted_selected()
     2. If sel: take the last one, read obj.bounding_box (.left, .bottom),
     return (bbox.left, bbox.bottom + GAP) where GAP = 0.5.
     3. Fallback (no selection): return (0.0, 0.0).

     Insertion:
     otype = dia.get_object_type(type_name)
     obj, _, _ = otype.create(x, y)
     _apply_label(obj, label)  # sets name/text if property exists
     diagram.active_layer.add_object(obj)
     diagram.update_extents()
     diagram.flush()

     Action 2 — InsertTextBoxList (menupath 
     /DisplayMenu/Objects/InsertTextBoxList)

     Opens a Gtk.Window containing a Gtk.TextView inside a Gtk.ScrolledWindow
     and a "Insert" button. On confirm:
     1. Split buffer text on newlines; strip empties.
     2. Compute start (x, y) from the same _placement_point helper.
     3. For each word, create a Flowchart - Box, set obj.properties["text"] = 
     word, add to diagram.active_layer, then advance y by a fixed row height
     (e.g. 1.2 units) for the single-column layout.
     4. diagram.update_extents(); diagram.flush().

❯ how do i use this plugin?                                                     

● Quick start

  1. Confirm it loaded. Launch Dia, open (or start) a diagram. In the diagram
  window's menubar, open the Objects menu — you should see two new entries near
  the top:
  - Quick Insert Shape...
  - Insert Text Boxes From List...

  If they're missing, Dia didn't pick up the plugin — check that the file is at
  ~/.dia/python/keyboard_insert.py and restart Dia.

  2. Quick Insert Shape (single shape).

  - Objects → Quick Insert Shape…
  - Shape field: start typing — e.g. class, dec, proc, actor, usecase. A
  dropdown shows matches from UML + Flowchart. ↓ to pick, Enter to accept — the
  field fills with the full type name (UML - Class, Flowchart - Decision, …).
  - Label field: optional. Whatever you type becomes the shape's name (UML) or
  text (Flowchart) property.
  - Enter (or click Insert) to place. Esc closes the dialog.
  - Where it lands: directly below whatever object is currently selected. If
  nothing's selected, it lands at (0, 0). Trigger again with the just-placed
  shape selected → next shape stacks below it.

  3. Insert Text Boxes From List (many boxes at once).

  - Objects → Insert Text Boxes From List…
  - Paste or type one word/phrase per line into the big text area.
  - Ctrl+Enter (or click Insert) → one Flowchart - Box per non-blank line,
  stacked in a single vertical column, each labelled with that line.
  - Same placement rule: the column starts below the selected object.

  4. Bind keys (optional but the point of the plugin).

  Edit ~/.config/dia/menurc — create it if it doesn't exist — and add:

  (gtk_accel_path "<Actions>/AllActions/QuickInsertShape"  "F5")
  (gtk_accel_path "<Actions>/AllActions/InsertTextBoxList" "F6")

  Restart Dia. F5/F6 now open the two dialogs. Pick any keys you like — the
  syntax is GTK's, e.g. "<Control>i", "<Shift>F2".

  If those exact accel paths don't fire, use Dia's live rebinding: hover the
  menu item, press your desired key combo — GTK writes the correct path to
  menurc on exit. On next launch it'll auto-load.

  Typical workflow

  - Building a flowchart from a brainstorm: dump 20 lines into Insert Text 
  Boxes, then hand-arrange and connect. Faster than clicking 20 palette buttons.
  - Sketching a UML class hierarchy: F5 → class → name it Order → Enter. F5 →
  class → LineItem → Enter. Each new class stacks below the previous one; drag
  to spread out and add relations.
  - Mixed diagram: F5 anytime to grab a Decision, Actor, Component, whatever —
  no palette hunting.

  Troubleshooting

  - Menu items don't appear: file isn't at ~/.dia/python/keyboard_insert.py, or
  Dia's Python plugin loader is disabled. Run Dia from a terminal with dia 
  --log-to-stderr and look for import errors mentioning keyboard_insert.
  - Dialog fails to open, error toast says "PyGObject (gi) is not available":
  your Dia's embedded Python doesn't have PyGObject. If you built Dia yourself,
  install python3-gi (or the equivalent for your distro) into whichever Python
  that build links against.
  - "No UML/Flowchart shape matches 'foo'": try broader terms — the match is
  substring against both the friendly name and the Sheet - Type name.

● Yes — but there's a Nix-specific catch. Dia embeds a specific Python from the
  store (I see python3-3.13.12 in its closure), so just doing
  environment.systemPackages = [ python3Packages.pygobject3 ] won't help — Dia
  doesn't consult your user PYTHONPATH by default, and even if it did,
  pygobject3 needs matching GI_TYPELIB_PATH for GTK3's typelibs at runtime.

  Two ways to fix it, from easiest to cleanest:

  Option A — Wrapper (drop into your NixOS config)

  { pkgs, ... }:
  let
    py = pkgs.python313.withPackages (ps: [ ps.pygobject3 ]);
    diaWithGi = pkgs.symlinkJoin {
      name = "dia-with-gi";
      paths = [ pkgs.dia ];
      nativeBuildInputs = [ pkgs.makeWrapper ];
      postBuild = ''
        wrapProgram $out/bin/dia \
          --prefix PYTHONPATH : "${py}/${py.sitePackages}" \
          --prefix GI_TYPELIB_PATH : "${pkgs.gtk3}/lib/girepository-1.0" \
          --prefix GI_TYPELIB_PATH : "${pkgs.glib.out}/lib/girepository-1.0" \
          --prefix GI_TYPELIB_PATH : "${pkgs.gdk-pixbuf}/lib/girepository-1.0" \
          --prefix GI_TYPELIB_PATH : "${pkgs.pango.out}/lib/girepository-1.0" \
          --prefix GI_TYPELIB_PATH : "${pkgs.harfbuzz.out}/lib/girepository-1.0"
   \
          --prefix GI_TYPELIB_PATH : "${pkgs.atk}/lib/girepository-1.0"
      '';
    };
  in {
    environment.systemPackages = [ diaWithGi ];
  }

  python313 needs to match whatever version Dia was compiled against (currently
  3.13 in your closure). If nixpkgs bumps Dia to link a different Python, bump
  this too or the versions won't line up and import gi will still fail.

  If the nixpkgs dia package accepts python3 as an input (recent versions do),
  override it to pass a Python bundled with pygobject3:

  (pkgs.dia.override {
    python3 = pkgs.python313.withPackages (ps: [ ps.pygobject3 ]);

  Cleaner but only works if the derivation is written to accept that override —
  check pkgs.dia.override.__functionArgs or read its Nix expression.
